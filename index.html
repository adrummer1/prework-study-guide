HTML

  • The head element contains information about the webpage
  • The body element represents the visible content shown to the user
  • Within the head element are meta elements not viewable to the user
  • Within the body element are elements for structuring content, including elements for headings, plain text, images and lists

CSS

  • A margin indicates how much space we want around the outside of an element.
  • A padding indicated how much space we want around the content inside an elemnent.
  • The color property can be assigned in multiple ways, including semantic wording and hex codes.
  • The border property can be used to assign weight to a border around an element.
  • A class can be used to assign CSS properties to a segment of specified HTML elements.
  • CSS can be used to style content in a variety of ways, which include inline CSS, internal CSS, and external CSS.

Git

  • Git status: Checks which branch we are currenty working on
  • Git checkout -b branch-name: Creates a new branch and switches to it
  • Git commit: saves changes in a repository to new commit
  • Using ^ or ~: Using ^ to move up one commit at a time or ~number to move a number of times
  • Git merge: merges commit with two parents
  • Git rebase: combines work between branches

JavaScript

  • A variable is a named container that allows us to store data in our code.
  • Control flow is the order in which a computer executes code in a script.
  • JavaScript files are read from top to bottom, which is important for control flow.
  • A function is a set of instructions to tell the computer how to perform a certain task.
  • You must call a function by name in order for it to execute it.
  • A functions name is always followed by parenthesees.
  • Use the consol log to add text to the console that explains the function.
  • We used the JS built-in Math object to generate a randome topic.
